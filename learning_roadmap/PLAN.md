## 模块 A：接口协议

### A1. gRPC
- **学什么**：Protobuf 序列化、.proto 文件编写、4 种 RPC 模式（unary/server-stream/client-stream/bidi）
- **实操**：写一个 Python gRPC 服务 → 用 grpcurl 调试 → 写测试脚本
- **面试点**：gRPC 和 REST 有什么区别？怎么测 gRPC 接口？

### A2. WebSocket
- **学什么**：HTTP 升级握手、长连接、心跳机制
- **实操**：写一个聊天服务 → Python websockets 客户端测试
- **面试点**：WebSocket 和 HTTP 长轮询有什么区别？怎么测并发连接？

## 模块 B：中间件速览

### B1. Redis — 内存缓存
- **一句话**：把热数据放内存，读写飞快
- **实操**：docker 跑 Redis → set/get → 验证过期 → pub/sub
- **测开关注**：缓存穿透/击穿/雪崩是什么？怎么模拟测试？

### B2. Kafka — 消息队列
- **一句话**：生产者发消息，消费者收消息，中间 Kafka 保证不丢
- **实操**：docker-compose 跑 Kafka → 生产/消费 → 验证消息顺序/不丢
- **测开关注**：消息积压怎么发现？消息丢失怎么验证？

### B3. Elasticsearch — 搜索引擎
- **一句话**：建倒排索引，毫秒级全文搜索
- **实操**：docker 跑 ES → 写入文档 → 全文搜索 → 聚合查询
- **测开关注**：怎么写测试验证搜索结果？写入后多久能搜到？

### B4. MinIO — 对象存储
- **一句话**：S3 兼容的文件存储，存图片/视频/日志
- **实操**：docker 跑 MinIO → 上传下载 → presigned URL
- **测开关注**：文件完整性（MD5）、并发上传怎么测？

## 模块 C：微服务体系

### C0. ZooKeeper — 分布式协调服务
- **一句话**：分布式系统的"协调员"——服务注册、配置管理、分布式锁、Leader 选举
- **核心概念**：ZNode 树形结构（类似文件系统）、Watcher 监听机制、ZAB 一致性协议、过半存活
- **面试点**：为什么 ZK 节点数要是奇数？ZK 和 Nacos/Etcd 有什么区别？
- **典型用途**：Kafka（老版本依赖 ZK）、Dubbo（服务注册）、HBase

### C1. Nacos — 注册中心 + 配置中心
- **一句话**：服务注册到 Nacos，其他服务从 Nacos 发现它；配置也统一在 Nacos 管
- **概念**：服务注册/发现、健康检查、配置热更新、命名空间
- **测开关注**：怎么验证服务注册成功？配置变更后服务是否生效？

### C2. Gateway — API 网关
- **一句话**：所有请求的统一入口，负责路由、鉴权、限流
- **概念**：路由规则、过滤器链、限流策略（令牌桶/漏桶）
- **测开关注**：怎么测试限流规则？路由错误的请求去哪了？

### C3. 微服务全景
- 服务注册发现(Nacos) → API 网关(Gateway) → 服务间调用(RPC) → 熔断降级(Sentinel)
- **实操**：k3s 上部署 2~3 个微服务互相调用（概念层面）
- **面试点**：微服务和单体有什么区别？服务雪崩怎么解决？

## 模块 D：大数据 / 流计算

> ⚠️ Flink、Doris、DataBus 对资源要求高（至少 4G+ 内存），这台 2C2G 机器跑不动。  
> ZooKeeper、Nacos、Kafka（轻量模式）可以在本机 docker 跑。  
> **学习方式**：能实操的实操，跑不动的用概念 + 架构图 + 面试话术。

### D1. Flink — 流式计算引擎
- **一句话**：数据像水流一样持续进入，Flink 实时计算（统计/聚合/报警），不停机
- **核心概念**：Source → Transform → Sink、Event Time vs Processing Time、Checkpoint（容错）
- **对比**：Spark Streaming（微批处理）vs Flink（真正的逐条流处理）
- **面试点**：Flink 怎么保证 exactly-once？Checkpoint 原理？

### D2. Doris — OLAP 分析型数据库
- **一句话**：面向报表/大屏的列式存储数据库，查百亿行数据秒级返回
- **核心概念**：列式存储、MPP 架构、前缀索引、物化视图
- **对比**：MySQL（OLTP 一行行事务）vs Doris（OLAP 大批量分析）
- **面试点**：OLTP 和 OLAP 的区别？Doris 为什么查询这么快？

### D3. DataBus — 数据总线 / 数据同步
- **一句话**：把 MySQL 的数据变更（binlog）实时同步到 Kafka / Doris / ES
- **常见方案**：Canal（阿里）→ 监听 MySQL binlog → 发到 Kafka → Flink 处理 → 写入 Doris
- **数据流**：`MySQL → Canal/Kafka → Flink → Doris/ES/Hive`
- **面试点**：怎么保证数据同步不丢？数据同步延迟怎么监控？

## 推荐学习顺序

```
A1(gRPC) → B1(Redis) → B2(Kafka)
    → A2(WebSocket) → B3(ES) → B4(MinIO)
    → C0(ZooKeeper) → C1(Nacos) → C2(Gateway)
    → D1(Flink) → D2(Doris) → D3(DataBus)
    → C3(微服务全景串联)
```

先基础协议 → 中间件 → 微服务组件 → 大数据链路，最后用 C3 把所有东西串起来形成体系。
